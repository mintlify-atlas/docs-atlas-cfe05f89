TruequeU is a single-page application (SPA) with no backend. All application state is managed on the client using Zustand and persisted to the browser's `localStorage`. There are no API calls, no server, and no database вЂ” the app is entirely self-contained in the browser.

<Note>
  There is no backend API. All data вЂ” listings, user sessions, and chat messages вЂ” lives in the browser via `localStorage`. Clearing browser storage resets the app to its seed data state.
</Note>

## Application Entry Point

The app is bootstrapped in `main.tsx`, which mounts React into the `#root` DOM node and wraps everything in `BrowserRouter` to enable client-side routing:

```tsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import './index.css'
import App from './App.tsx'
import { BrowserRouter } from 'react-router-dom'

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>,
)
```

## Data Flow

The app follows a unidirectional data flow with no async effects or network layer:

<Steps>
  <Step title="User action">
    The user interacts with a page component (e.g., submits a listing form, clicks favorite, logs in).
  </Step>
  <Step title="Store action called">
    The component calls an action from `useStore` (e.g., `addListing`, `toggleFavorite`, `login`).
  </Step>
  <Step title="Zustand updates state">
    Zustand runs the action's `set()` call, producing a new immutable state snapshot.
  </Step>
  <Step title="Persist middleware syncs to localStorage">
    The `persist` middleware automatically serializes the `listings`, `user`, and `chats` slices to `localStorage` under the key `marketplace-storage`.
  </Step>
  <Step title="React re-renders">
    All components subscribed to the changed state slice re-render with the new data.
  </Step>
</Steps>

## Vite Configuration

The `vite.config.ts` is minimal вЂ” TailwindCSS and React Fast Refresh are registered as plugins:

```ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
})
```

## Services Layer

The `services/` directory contains pure functions for business logic. These functions do not touch the Zustand store directly вЂ” they are called by page components which then dispatch the result into the store.

| Service | Responsibility |
|---|---|
| `auth.service.ts` | Login validation, user registration, email uniqueness check, password rules |
| `listing.service.ts` | Validates listing fields (title length, description length, price range) |

Seed users are stored in `data/users.ts` and checked during login before falling back to `localStorage`-registered accounts.
